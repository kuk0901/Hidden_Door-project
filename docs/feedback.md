## 느낀점

- 조병철:

```
팀원들과 같이 React를 공부해 프로젝트에 적용하면서 다양한 기능을 직접 다뤄볼 수 있었고, 왜 UI 라이브러리를 사용하는지 그 장점을 몸소 느낄 수 있었다. 
그리고 MongoDB를 사용하면서 RDBMS와의 차이점을 보다 명확하게 이해할 수 있었다. 

학원에서 배운 기술 외에도 스스로 학습한 내용을 실제 프로젝트에 활용해 보면서, 새로운 기술을 익히는 과정이 어렵지만 동시에 큰 성취감을 준다는 것을 깨달았다. 
앞으로도 이런 과정을 반복하면서 조금씩 역량을 넓혀가고 싶다.
```

<br />

- 김진우:

```
React에 대해 공부를 하며 async/await 라는 문법을 새롭게 배우게 되었는데 이전에 배웠던 메서드 체이닝 형태의 비동기 요청과는 달리 
요청이 완료되기 전까지는 다음 단계가 실행되지 않아 동작 순서를 명확하게 체감할 수 있었다.

또한, MongoDB를 처음 사용하면서 RDBMS와의 차이점을 이해하기 위해 참조키를 사용하지 않고 데이터를 직접 도큐먼트에 넣고 
이를 인덱스 처리하는 과정을 반복하며 데이터를 검색하는 방법으로 인덱스를 설정하는 것이 
조회 성능에 있어서 얼마나 중요한 일인지를 깨달을 수 있었다.

새로 배운 언어들을 사용하는 과정이 쉽지는 않았지만, 팀원에게 물어보고 직접 사용해보며 기술의 작동 원리와 적용 방법을 이해하게 되었다. 
앞으로도 새로운 언어에 도전하며 성장할 수 있도록 노력하겠다.
```

<br />

- 국하현:

```
주제가 방탈출 예약 카페 웹사이트다 보니, 예약과 관련된 로직을 구현하는 과정에서 부적절한 예약 시도를 방지하는 기능에 대해 팀원들과 많은 고민을 했다. 
URL 직접 접근이나 JavaScript를 통한 예외적인 요청을 막기 위해 다양한 방법을 적용해 보면서 웹 서비스 보안의 중요성을 다시 한 번 깨달았다.

특히 JWT를 활용해 액세스 토큰과 리프레시 토큰을 어떻게 관리할지 고민하고 구현하는 과정에서, 
웹 보안에 대해 실질적으로 배울 수 있었던 점이 매우 좋았다.

학원에서 배운 내용이 아니라 직접 공부한 기술로 프로젝트를 진행하다 보니 개발 과정이 쉽진 않았지만 오히려 그만큼 성장할 수 있었고, 
완성된 결과물에 대한 만족감도 더욱 컸던 것 같다.
```
